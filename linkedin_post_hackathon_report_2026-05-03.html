Retrieval Pipeline Deep Dive

The hybrid retrieval system combines lexical and semantic signals for robust document ranking.

sequenceDiagram participant T as Ticket Query participant D as Domain Filter participant B as BM25 Engine participant S as Semantic Hash participant R as RRF Fusion participant O as Output T->>D: Parse domain signal D->>D: Keyword classification D->>B: Filtered corpus D->>S: Filtered corpus B->>B: Build DF/IDF indexes B->>R: Lexical rankings S->>S: Trigram → 512D mapping S->>R: Semantic rankings R->>R: score = 1/(k+rank_bm25) + 1/(k+rank_sem) R->>O: Final ranked chunks O->>O: Dynamic top-K selection

Semantic hashing note: The semantic proxy uses hashed character trigrams mapped to 512D space — not a neural embedding model, but a deterministic scheme that provides meaningful similarity signals without any ML dependencies.

# RRF Fusion Formula
score = 1 / (k + rank_bm25) + 1 / (k + rank_semantic)
# where k = 60, weights = [1.0, 1.0]

# Dynamic context selection
if confidence >= 0.75:  top_k = 4
elif confidence >= 0.55: top_k = 6
else:                     top_k = 8

Multi-Layer Fallback Strategy

Reliability was non-negotiable. Our system implements five layers of fallback to ensure every ticket is processed correctly.

Fallback Layers

  1. Deterministic Default — Entire pipeline runs without any LLM using template-based response assembly
  2. Circuit Breaker — Opens after 3 consecutive failures, 60-second cooldown
  3. Bounded Retries — Up to 2 retries with exponential backoff (base 0.35s) + 8s timeout
  4. Validation Gate — Strict JSON schema validation with denylist/allowlist checks
  5. A/B Calibration — Gemini only enabled if measurable gain + zero safety regression

Result: After A/B calibration, the system chose deterministic mode (threshold: 0.66, weighted score: 0.7278). No measurable gain from Gemini on this dataset.

Transparency & Learnings

Bug disclosure: A bug in the pipeline resulted in all reports being incorrectly marked as escalated. This means our submission likely won't place — but we're owning it transparently.

What We Could Have Done More

With more time, we could have improved the architecture significantly — added better chunking strategies, integrated real embedding models, expanded the escalation taxonomy, and refined the confidence calibration. But within a 24-hour constraint, we shipped what we could and are proud of that.

Learning > Winning

The experience, the late-night debugging sessions, the deep dive into building RAG systems from scratch, and understanding the tradeoffs between deterministic and LLM-driven pipelines — that learning counts far more than any ranking on a leaderboard.

How This Was Built

Most of the implementation code was written by AI agents — Codex and OpenCode. Our team's focus was on:

  • Agent orchestration — coordinating multiple AI agents effectively
  • Architecture design — making high-level engineering decisions about pipeline structure, fallback strategies, and safety gates
  • Review and iteration — validating outputs, catching edge cases, and steering the agents toward correct implementations

This is what modern AI-assisted development looks like. The human role shifts from writing every line of code to orchestrating, architecting, and quality-assuring.
